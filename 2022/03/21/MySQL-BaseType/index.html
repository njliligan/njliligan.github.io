数据类型

数据基于MySQL8.0

数值类型

注意事项

1，对于整数类型M表示最大显示宽度，为255，显示宽度与储存值的大小无关。所以整数类型上的M实际上是指的填充多少位，例如TINYINT(1),和TINYINT(4)，如果对于TINYINT(4)来说，实际值是2，那么输出的时候就会填充为0002
2，对于浮点和定点数据类型，M是可以存储的总位数。
3，如果使用ZEROFILL指定数字列，MySQL会自动将UNSIGNED属性添加到该列8.0.17弃用)，可以使用LPAD()填充宽度，或者格式化后储存在CHAR中。
4，UNSIGNED类型可以存入SIGNED类型。
5，从8.0.17开始UNSIGNED弃用了对FLOAT，DOUBLE，DECIMAL以及任何同义词的支持，可以使用简单的CHECK约束
6，SERIAL 别名-> BIGINT UNSIGNED NOT NULL AUTO_INCREMENT UNIQUE
7，SERIAL DEFAULT VALUE 在INTEGER中定义的别名是 NOT NULL AUTO_INCREMENT UNIQUE

语法

1，BIT[(M)]
M表示每个值的位数，从1到64，M省略时为1.
2，TINYINT[(M)] [UNSIGNED] [ZEROFILL]
有符号是-128到127，无符号是0到255.
3，BOOL，BOOLEAN
同义词是TINYINT(1)，0表示FALSE，非0认为是TRUE.

但是反过来，对于FALSE和TRUE来说，他们分别只对应0和1。
4，SMALLINT[(M)] [UNSIGNED] [ZEROFILL]
5，MEDIUMINT[(M)] [UNSIGNED] [ZEROFILL]
6，INT[(M)] [UNSIGNED] [ZEROFILL]
7，INTEGER[(M)] [UNSIGNED] [ZEROFILL]
是INT的同义词
8，BIGINT[(M)] [UNSIGNED] [ZEROFILL]
大整数
SERIAL 别名-> BIGINT UNSIGNED NOT NULL AUTO_INCREMENT UNIQUE
注意事项
1)，所有的算术运算都是使用BIGINT或者DOUBLE值完成的，所以不应该使用大于63位的无符号大整数，
2)，以下的情况MySQL可以处理BIGINT
(1)，使用整数在BIGINT列中存储大的无符号值。
(2)，在MIN(col_name)或者MAX(col_name)，当col_name行的类型为BIGINT时。
(3)，当使用(+,-,*,…..)等操作的时候。
(4)，可以使用String来存储BIGINT的值。
(5)，两个BIGINT计算，注意溢出。
9，DECIMAL[(M[,D])] [UNSIGNED] [ZEROFILL] M表示总的位数，D，表示小数点后面位数的精度，-号不计入M中，如果D为0，那么就没有小数部分，M最大值为65，D的最大值为30，D未定义时候为0，M未定义时为10。UNSIGNED在8.0.17开始弃用。DECIMAL列的基本运算 (+, -, *, /)都是以65为精度完成的。
10，DEC [(M[,D])] [UNSIGNED] [ZEROFILL]，
NUMERIC[(M[,D])] [UNSIGNED] [ZEROFILL]，
FIXED[(M[,D])] [UNSIGNED] [ZEROFILL]
均是DECIMAL的同义词
11，FLOAT [(M[,D])] [UNSIGNED] [ZEROFILL]
M是总位数，D是小数点后的位数。如果省略M和D，则将值存储在硬件允许的范围内。单精度浮点数精确到大约 7 位小数。FLOAT[M,D]从8.0.17开始弃用，UNSIGNED从8.0.17开始也已经弃用。
12，FLOAT(P)
一个比较小浮点数，P表示以位为单位的精度，但是MySQL仅仅用这个来判断是使用FLOAT还是DOUBLE来生成数据，如果p是从 0 到 24，则数据类型变为FLOAT无M或 D值。如果 p是从 25 到 53，则数据类型变为DOUBLE无 M或D 值。UNSIGNED，如果指定，则不允许负值。从 MySQL 8.0.17 开始，该 UNSIGNED属性已被类型列FLOAT（和任何同义词）弃用，您应该期望在 MySQL 的未来版本中删除对它的支持。考虑对此类列使用简单的CHECK约束。
13，DOUBLE [(M[,D])] [UNSIGNED] [ZEROFILL]
一个正常大小的浮点数，M是总位数，D是小数点后的位数。如果省略M 和D，则将值存储在硬件允许的范围内。双精度浮点数精确到大约 15 位小数。DOUBLE和UNSIGNED均在8.0.17版本中弃用。
14，DOUBLE PRECISION[(M[,D])] [UNSIGNED] [ZEROFILL] ，REAL[(M[,D])] [UNSIGNED] [ZEROFILL]
都是DOUBLE的同义词，如果”REAL_AS_FLOAT” SQL模式是开启状态，那么比起DOUBLE来说，REAL更像是FLOAT的同义词.

日期和时间

语法

表示日期的有五个DATE，TIME，DATATIME，TIMESTAMP和YEAR。
对于DATE和DATATIME范围，表示早期的值可能有效，但是不能保证。
在MySQL中，TIME，DATETIME和TIMESTAMP的值最多到微秒级别(六位小数)，常用的语法为type_name(fsp)，其中type_name代表TIME，DATETIME和TIMESTAMP，fsp是秒上的小数精度

DATETIME和TIMESTAMP都具有自动更新值的属性。
1，DATE
值从’1000-01-01’` to到’9999-12-31’，MySQL对于DATE展示类型为‘YYYY-MM-DD’，但是也允许使用Strings和numbers
2，DATETIME[(fsp)]，
时间和日期，范围是从1000-01-01 00:00:00.000000到 9999-12-31 23:59:59.999999，默认格式化类型为 YYYY-MM-DD hh:mm:ss[.fraction]，如果fsp未定义，那么fsp默认为0。
3，TIMESTAMP[(fsp)]
范围从1970-01-01 00:00:01.000000UTC 到2038-01-19 03:14:07.999999 UTC，不能表示1970-01-01 00:00:00，因为他要求这个时候是0，但是0在这里有一个默认的值为0000-00-00 00:00:00。另外TIMESTAMP的值会随着服务器的时区改变，取决于explicit_defaults_for_timestamp这个系统变量。
explicit_defaults_for_timestamp是启用的情况下，DEFAULT CURRENT_TIMESTAMP 或者 ON UPDATE CURRENT_TIMESTAMP 在任何TIMESTAMP列上将不会自动赋值。
explicit_defaults_for_timestamp是关闭的的情况下，除非另有指定，否则表中的第一个TIMESTAMP列被定义为自动设置最近修改的日期和时间(如果没有显式地指定值)。这使得TIMESTAMP用于记录INSERT或UPDATE操作的时间戳非常有用。您还可以通过为任何TIMESTAMP列分配一个NULL值来将其设置为当前日期和时间，除非它已经用NULL属性定义以允许NULL值。
可以使用TIMEDEFAULT CURRENT_TIMESTAMP和ON UPDATE CURRENT_TIMESTAMP定义子句指定自动初始化和更新到当前日期和时间。默认情况下，第一列具有这些属性。但是，一个表中所有TIMESTAMP行都可以具有这些属性。
4，TIME(fsp)
范围从-838:59:59.000000 到 838:59:59.000000，展示为hh:mm:ss[.fraction]
fsp默认为0，范围为0-6。
5，YEAR[(4)]
范围为1901到2155，或者0000，格式化为YYYY。另外，8.0以前两位数的年份会被弃用，对于在DATETIME，DATE，TIMESTAMP类型，年份转换使用这些规则：在00-69之间的，会变成2000-2069，在70到99之间的，会变为1970-1999。对于YEAR来说规则也是相同的，但是也有例外，例如，数字00插入YEAR是0000而不是2000，如果要指定是2000，必须要插入’0’或者’00’，
SUM()和AVG()不适用于日期。
